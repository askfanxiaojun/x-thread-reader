隐私政策

你的 Thread 内容，默认只留在浏览器里。

X Thread Reader 采用 local-first 设计。下面这页会说明插件会处理哪些网站内容、为什么需要这些权限， 以及这些数据如何被限制在插件公开披露的单一用途之内。

生效日期：2026 年 4 月 2 日 适用于官网与插件

隐私概览

X Thread Reader 会读取你当前已经打开的 X/Twitter 页面中的 Thread 内容， 目的是在你主动触发时提取当前线程，并整理为 Markdown，方便你个人保存、阅读或归档。

X Thread Reader 不提供远程解析服务，不要求注册账号，也不会在正常使用中把你导出的 Thread 内容上传到我们自己的服务器。

单一用途

这个插件只有一个主要用途：当你在 X/Twitter 推文页面主动点击插件时，它提取当前 Thread， 并把结果转换为你可以本地复制或下载的 Markdown。

插件会处理哪些数据

  • 你主动触发提取时所在的 X/Twitter 当前标签页。
  • 该线程页面中用于导出的站点内容，包括推文文本和图片链接。
  • 只有在你点击复制时，才会调用浏览器剪贴板能力。

为什么需要这些权限

  • `activeTab`：仅在你点击插件后使用，用来读取当前标签页并提取你请求导出的 Thread。
  • `x.com` 与 `twitter.com` 主机权限：仅用于让插件在支持的 X/Twitter 线程页面上运行提取逻辑。

数据如何被使用

X Thread Reader 访问到的网站内容，仅用于完成你主动请求的 Thread 提取和 Markdown 导出。 我们不会把这些内容用于广告投放、用户画像、与插件单一用途无关的分析，或转售给第三方。

我们不会收集或出售什么

  • 不会要求账号注册。
  • 不会在服务器保存你的 Thread 历史。
  • 不会基于你导出的内容做个性化分析。
  • 不会出售个人数据或网站内容。
  • 不会将提取内容用于广告或数据经纪用途。

导出文件如何处理

无论是复制 Markdown 还是下载 `.md` 文件，结果都在你的浏览器会话中本地生成。 文件后续如何保存、同步或导入笔记工具，由你自己的设备和工作流决定。

数据共享与传输

在正常使用中，提取到的 Thread 内容不会被传输到我们的服务器。我们也不会为了与插件单一用途无关的目的， 把这些内容出售、共享或转移给第三方。

数据保留

因为提取过程发生在本地浏览器中，所以除非你自己选择复制或下载结果，否则这些 Thread 内容只会停留在你的浏览器会话里。

官网统计

当前官网是一个静态站点。如果未来增加统计脚本，应该尽量采用隐私友好的方案， 并且不采集 Thread 导出内容。如果这部分发生变化，这页需要同步更新。

与 Chrome Web Store 政策的对齐

X Thread Reader 对所访问数据的使用，旨在符合 Chrome Web Store User Data Policy， 包括 Limited Use 要求，并且仅限于插件已披露的单一用途。

隐私政策更新

如果未来加入云端功能、账号系统、服务端处理逻辑，或更广泛的数据分析能力， 这份隐私政策应该在功能上线前更新。

快速结论

  • 本地提取
  • 无需登录
  • 仅为导出当前 Thread 而处理页面网站内容
  • 不在第三方服务器保存 Thread
  • Markdown 在客户端生成
