为什么这个工具有价值

你不是想“截图保存推文”，你是想把 Thread 真正存下来。

很多高质量 Thread 适合长期阅读、整理观点、做资料归档。X Thread Reader 的目标， 就是把这种“值得留住的内容”从时间线里提出来。

完整

自动滚动加载更多内容，避免长 Thread 只抓到一半。

干净

结果整理成 Markdown，后续放进 Obsidian、Notion、文档或仓库都很自然。

本地

提取和格式化都在浏览器内完成，不依赖远程解析服务。

核心功能

它不是大而全工具，而是专门解决“导出 Thread”这件事。

对搜索用户来说，最重要的是快速知道：能不能完整抓、能不能导出、能不能放心用。 这几个点你的网站都应该讲清楚。

01

自动识别 Thread 页面

进入包含 `/status/` 的推文页面后，插件会判断是否可以开始提取。

02

长线程自动滚动抓取

针对内容较长的 Thread，插件会继续滚动加载，尽量把内容收完整。

03

保留图片链接

导出的 Markdown 会附带图片链接，适合后续归档、整理和引用。

04

一键复制或下载

提取完成后，可以直接复制 Markdown，或下载成 `.md` 文件保存。

使用方法

三步完成一次 Thread 导出。

流程越短，越像一个顺手的阅读工具，而不是一个复杂平台。

1

打开一条 X / Twitter 推文

访问任意 `x.com` 或 `twitter.com` 下带 `/status/` 的页面。

2

点击浏览器工具栏里的插件

弹窗会开始提取内容，并实时显示进度。

3

复制 Markdown 或下载文件

把结果放进你的笔记系统、资料库或写作工作流里。

常见问题

用户在安装前最关心的，官网里都应该直接回答。

比如：支不支持 x.com、会不会上传数据、导出内容是否包含图片、现在去哪里安装。 这些问题回答得越清楚，转化越高。

支持 x.com 和 twitter.com 吗？

支持。只要是标准的推文详情页，插件就可以识别并尝试提取 Thread。

会把我保存的内容上传到服务器吗？

不会。提取和格式化都在浏览器本地完成，结果由你自己复制或下载。

导出内容会保留图片吗？

会。导出的 Markdown 会包含图片链接，方便后续整理。
