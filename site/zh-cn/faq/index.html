FAQ

用户在第一次使用前，通常会先问这些问题。

这里集中回答关于兼容性、安装方式、导出结果和隐私的常见问题。 对 SEO 来说，这类页面也很适合承接明确搜索意图。

X Thread Reader 是做什么的？

它可以完整提取 X/Twitter 上的 Thread，并导出为适合保存和整理的 Markdown 文件。

支持 x.com 和 twitter.com 吗？

支持。只要是标准的推文详情页，并且 URL 中带 `/status/`，就可以识别。

能处理很长的 Thread 吗？

可以。这正是插件的核心能力之一：会自动滚动并尽量把后续内容加载出来。

导出内容会保留图片吗？

会。Markdown 结果中会保留图片链接，方便你后续继续整理和引用。

会把我提取的内容上传到服务器吗？

不会。提取和格式化都在浏览器本地完成，结果由你自己复制或下载。

现在在哪里安装？

Chrome Web Store 页面还在准备中。当前可以先通过 GitHub 仓库查看源码和手动安装方式。

安装与使用说明

当前安装方式

目前项目以开源 Chrome 插件的形式存在。你可以克隆仓库、构建项目， 再在 Chrome 里以开发者模式加载已解压扩展程序。

等 Chrome Web Store 页面正式上线后，只需要把首页主按钮替换成商店安装链接即可。

典型使用场景

  • 把研究类 Thread 留存下来，避免之后很难再翻到。
  • 把长推文导入 Obsidian、Notion 或本地知识库。
  • 保存含图较多的教程线程，方便后续写作和引用。

什么时候可能失败

如果 X/Twitter 页面结构发生明显变化，或者内容受限，提取可能受影响。 但因为项目是开源的，选择器变化后可以快速修复。
